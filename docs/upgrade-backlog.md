# 향후 고도화 백로그 (현재 Phase에서는 구현하지 않음)

마스터 프롬프트 §14 기준. 클론 구조 설계 시 아래 항목이 들어올 자리를 막지 않는 것만 고려한다.

## 검색/추천
- AI 기반 호텔 추천
- 자연어 호텔 검색
- 개인화 추천
- 객실 추천
- 항공과 호텔 결합 검색 고도화
- 지도 기반 검색 (원본에 "지도에서 보기" 진입점 존재 — 클론 Phase 5에서 UI 자리만)

## 사용자 편의
- 호텔 비교 고도화 (원본에 "비교하기" 존재)
- 최근 검색 자동 저장 고도화 (원본: localStorage recentSearchHotels)
- 가격 변동 알림
- 예약 단계 간소화
- 여행 일정 생성
- 리뷰 요약
- AI 챗봇

## 회원/커머스
- 회원 등급
- 쿠폰 및 프로모션
- 소셜 로그인 실연동 (원본: 카카오/네이버/구글/애플)
- 다국어
- 다중 통화

## 운영/연동
- 관리자 CMS 연동 (배너/기획전/공지 — 원본은 /data/share/{date}/AV/I/ 업로드 경로 사용)
- ELLIS API 연동 (hotel.service 계약 재검증 필요 — docs/api-contract.md)
